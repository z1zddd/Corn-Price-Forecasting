🌽 玉米期货时序预测 · 策略规范 v1.0

Training / Inference / Testing 三阶段标准化 — 基于滑动窗口 (5td / 10td)

一、策略总览

756td
训练窗口 (约3年)
5td / 10td
预测目标 (短期收益)
Walk-Forward
验证方式

核心设计原则

  1. 时间顺序不可破坏 — 绝不随机打乱，训练数据必须在测试数据之前
  2. 回避数据泄露 — 训练窗口和预测目标之间必须有 gap（= horizon 天数），避免用未来信息训练
  3. 滑动窗口逐日推进 — 模拟真实交易场景：每天用最新可用数据重新训练，做一次预测
  4. 分层评估 — 区分回归误差(MAE/RMSE) 和方向胜率(Directional Accuracy)，后者对交易更有意义

二、滑动窗口定义

时间轴 ─────────────────────────────────────────────────────▶ 未来

|···················· 训练窗口 (756 trading days) ····················| |gap| |target|
|   t-761  ···  t-6 ··· t-5 | t-4 | t-3 | t-2 | t-1 |  TODAY  |···| t+5 |   ← 5td
                                                  ↑ 今天
|   t-766  ···  t-11 ··· t-10 | t-9 | t-8 | t-7 | ... | TODAY |···| t+10 |   ← 10td
                                                  ↑ 今天

"gap" = 预测窗口天数 (5 或 10)，确保训练不包含未来的价格信息

窗口参数表

参数值说明
train_window756 td约3年交易日，覆盖完整的市场周期
forecast_horizon5td / 10td短期预测，捕捉动量信号
gap= horizon训练截止日期 = 预测日期 - horizon，防泄漏
step1 td每天滚动一步，做一次预测
min_train_ratio90%如果有效训练样本少于 window 的 90%，跳过该步

三、训练策略 (Training Strategy)

3.1 数据划分

集合占比作用
Training滚动窗口 756td每次预测前，用最近756天数据训练
Validation内嵌于 walk-forward前一步的预测误差就是后一步的"验证"
Testing (回测)窗口之后的全部数据从第757天开始到数据集末尾，逐日生成预测记录

3.2 模型选择

模型类型适用
Ridge (α=5.0)回归基线，稳定，不易过拟合
GradientBoosting回归非线性特征交互
RandomForest回归鲁棒，处理缺失值
注：按 Wang&Zhang(2026)，农产品期货数据稀缺，优先用预训练基础模型(Time-MoE等)，从零训练的深度学习通常排名10-16位

3.3 Walk-Forward 训练流程

📊 取 [t-756, t-horizon] 数据 → 🧹 特征预处理 (impute + scale) → 🏋️ 训练模型 (Ridge/GBM/RF) → 🔮 预测 t 时刻收益 → 📝 记录 {pred, actual, signal} → ⏩ t=t+1，重复

3.4 特征工程规范

类别特征处理
价格动量dce_corn_close, dce_corn_ret_1d, rolling_mean/stdStandardScaler
基差corn_basis_cny_t, corn_basis_rate现货-期货价差
国际市场cbot_corn_close, cbot_wheat_corn_ratio, usd_cnyCBOT联动
天气ne_avg_precip_mm, ne_avg_t2m_c东北产区天气
资金面volume, open_interest量仓关系
分类变量season, month, weekdayOneHot(min_freq=10)
缺失值数值列中位数, 分类列众数SimpleImputer

四、测试/评估策略 (Testing Strategy)

4.1 回归指标

指标公式阈值
MAEmean(|y_true - y_pred|)越低越好
RMSE√(mean((y_true - y_pred)²))越低越好
R²1 - SS_res/SS_tot > 0 优于均值基线
Biasmean(y_pred - y_true)接近0

4.2 方向指标 (更重要!)

指标定义阈值
Directional Accuracy方向正确的比例 > 52%
Up Precision预测涨→实际涨的比例 > 50%
Down Precision预测跌→实际跌的比例 > 50%
Signal Ratio多头信号占比监测偏差

4.3 策略模拟指标

Equity Curve
(1+strategy_return).cumprod()
Max Drawdown
equity/peak - 1 的最小值
Sharpe-like
mean(strategy_ret)/std(strategy_ret)

策略规则：predicted_return > 0 → 做多 (+1); predicted_return < 0 → 做空 (-1)。按预测级别复利。

测试集神圣不可侵犯：回测期数据（walk-forward 预测日）绝不用于训练。每次 retrain 只用到 t-horizon 为止的数据。

五、推理/预测策略 (Inference Strategy)

5.1 最新预测流程

📅 今日 = 数据最后一天 → 📊 取 [today-756, today] 全部数据 → 🧹 预处理 (与训练同) → 🏋️ 全量数据训练 → 🔮 预测未来 5td / 10td → 📋 生成信号标签

5.2 信号分级

预测收益率信号交易建议
≥ +1.0%strong bullish积极做多
+0.3% ~ +1.0%mild bullish轻仓做多
-0.3% ~ +0.3%neutral观望
-1.0% ~ -0.3%mild bearish轻仓做空
≤ -1.0%strong bearish积极做空

5.3 多周期综合判断

5td 信号10td 信号综合策略
bullishbullish分阶段建多仓
bullishbearish减仓观望，等基差确认
bearishbullish短期对冲，保持低仓位
bearishbearish平多/做空，或对冲现货库存

6.3 Key Invariants (必须遵守)

  1. train_end = predict_date - horizon — 绝不能包含预测区间内的数据
  2. 特征工程 fit 只用训练集，transform 应用于所有集合
  3. 缺失值填充使用训练集统计量（中位数/众数），不穿越使用全局统计
  4. 分类变量 OneHot 使用 handle_unknown='ignore' + min_frequency=10 避免维度爆炸
  5. 每个 walk-forward step 必须重新 fit，不保留模型状态
  6. 5td 和 10td 两个 horizon 独立训练和评估，不共用模型
