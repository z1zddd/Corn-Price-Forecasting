# LEARNINGS — 模型对比回测

> 记录过程中的经验教训，供后续提炼。

## 经验 #1 — Codex token 上限 100M

**时间**: 2026-07-03
**场景**: 跨 50 小时的大会话，累计 ~99.94M tokens 后被强制中断
**教训**: GPT-5.5 在 Codex 中有 ~100M 总 token 限额。大任务需要拆成多个子会话，或让 Codex 用后台脚本 + 异步轮询减少 token 消耗
**可复用**: 是 — 大任务拆分会话策略

## 经验 #2 — Foundation model 零样本不适合短时序

**时间**: 2026-07-03
**场景**: TimePFN/Chronos-2/TiRex/Timer 在 12 个月回看 + 59 个窗口的 spike 分类任务上全部惨败
**教训**: 预训练时序模型在短序列（12个点）小样本场景下连 ML baseline 都打不过。不是因为模型不好，而是预训练数据分布的时序模式和我们任务不匹配
**可复用**: 是 — 选模型前先评估序列长度和样本量

## 经验 #3 — TimeXer 无预训练也能打

**时间**: 2026-07-03
**场景**: TimeXer 官方没有预训练权重，但从架构复现 + 微调后 AUC 0.884，比所有 foundation model 好
**教训**: 有内生/外生变量天然分区时，TimeXer 的 cross-attention 架构优于通用时序预训练模型
**可复用**: 是 — 多变量+内外生结构 → 优先考虑 TimeXer 类架构
